# Lucky Order - 出库数据分析平台

> 项目说明文档 / 2026-05-30 版本

---

## 一、项目背景

用户是**跨境电商履约人员（印章组供应链 BP）**，日常管理六大品类（胶垫、光敏、金属、亚克力、皮革、签字笔）的订单出库。原有工具（WPS / 飞书多维表格 / Power BI）各有痛点，因此自建网页端数据分析平台。

**核心需求**：
- 从马帮 ERP 导出 xlsx → 拖入网页 → 数据存云端
- 多人注册登录，数据隔离（各看各的）
- 专业数据看板 + AI 分析 + 新品追踪
- 自定义域名，无任何平台品牌水印

---

## 二、技术架构

| 层 | 技术选型 | 说明 |
|:---|:---|:---|
| **前端** | React + Vite | 最新技术栈，构建快 |
| **图表** | ECharts (echarts-for-react) | 专业数据可视化 |
| **数据库+登录** | Supabase | 免费额度大，自带邮箱注册/RLS 行级安全 |
| **AI 分析** | 本地计算引擎（无 API） | 不限次数永久免费 |
| **部署** | GitHub Pages + Actions CI/CD | 自动化构建部署 |
| **域名** | luckyck.cn（CNAME → GitHub Pages） | 海外服务器，无需 ICP 备案 |

## 三、已实现功能（✅ 全部完成）

### 📥 数据导入
- [x] **拖拽导入**：支持 .xlsx / .xls / .csv
- [x] **马帮格式直接识别**：自动检测马帮导出字段（订单编号、店铺名、SKU、国家、发货时间等）
- [x] **多 SKU 自动聚合**：同订单多行 SKU 自动合并为一条记录
- [x] **智能分类**：基于关键词规则（光敏→光敏类，金属→金属类等）自动推断产品品类
- [x] **字段映射预览**：导入前展示识别到的字段对应关系
- [x] **通用格式兼容**：非马帮格式也能自动识别字段名

### 📊 数据看板（全局筛选联动）
- [x] **全局筛选器**：时间范围（起止日期）、品类、供应商
- [x] **KPI 卡片**：总订单数、总出库量、总金额、加急单、品类数
- [x] **品类排行**：ECharts 柱状图（按出库量排序）
- [x] **品类占比**：环形饼图（TOP 8）
- [x] **近7天趋势**：折线图（每日单量）
- [x] **月度出库趋势**：折线图
- [x] **供应商排名**：柱状图
- [x] **订单状态分布**：饼图（待处理/生产中/已发货/已完成/已取消）

### ✨ 新品追踪
- [x] **添加新品**：输入品名 + 上线日期
- [x] **自动匹配出库数据**：从订单中匹配新品名称
- [x] **数据报告**：订单数、出库量、销售额、日均出库
- [x] **趋势图**：每日出库量折线图
- [x] **上线前预警**：如果上线前已有数据会自动标记 ⚠️

### 🧠 AI 洞察
- [x] **本地分析引擎**：不依赖任何外部 API
- [x] **分析报告内容**：整体概况、品类分析、趋势判断（上升/下降/平稳）、订单健康度、运营建议

### 🔧 工具功能
- [x] **智能分类规则**：管理关键词→品类映射（支持增删改、恢复默认）
- [x] **分享链接**：生成链接发给同事，对方可查看数据
- [x] **导出图片**：截取当前看板保存为 PNG
- [x] **海外营销日历**：精确到具体日期的跨境电商节日提醒

### 🔐 基础能力
- [x] **邮箱注册/登录**：无需邮箱确认，注册即用
- [x] **行级数据隔离（RLS）**：每人只能看到自己的数据
- [x] **luckyck.cn 域名上线**：GitHub Pages + CNAME
- [x] **CI/CD 自动部署**：代码推送 → Actions 自动构建 → 部署上线

## 四、数据结构（Supabase `orders` 表）

| 字段 | 类型 | 说明 |
|:---|:---|:---|
| id | BIGINT (PK) | 自增主键 |
| user_id | UUID | 关联 auth.users |
| created_at | TIMESTAMPTZ | 创建时间 |
| updated_at | TIMESTAMPTZ | 更新时间 |
| order_no | TEXT | 订单号 |
| product_category | TEXT | 产品品类（自动分类或手动指定） |
| product_name | TEXT | 产品名称 |
| quantity | INTEGER | 数量 |
| unit_price | DECIMAL(10,2) | 单价 |
| total_amount | DECIMAL(10,2) | 总金额 |
| order_status | TEXT | 状态：pending/processing/shipped/completed/cancelled |
| order_date | DATE | 下单/出库日期 |
| is_urgent | BOOLEAN | 是否加急 |
| supplier | TEXT | 供应商 |
| remark | TEXT | 备注（马帮导入时存店铺名和国家信息） |

## 六、待优化 / 下一步建议

### 🔴 高优先级（已有用户反馈）

1. **数据看板还不够完善** — 目前看板偏基础，可以加：
   - 更丰富的图表（热力图、散点图、雷达图）
   - 多维度交叉分析（品类×店铺、日期×品类）
   - 数据下钻（点击品类→查看该品类所有明细）
   - 环比/同比自动计算展示
   - 对比分析（选两个时间段对比）

2. **马帮导入后数据准确性验证** — 需要实际用马帮导出的数据测试一遍，确认字段映射、聚合逻辑、品类推断的准确性

### 🟡 中优先级

3. **店铺名/国家成为独立字段** — 目前存在 remark 备注里，建议数据库加 `store_name` 和 `country` 列，方便按店铺筛选和按国家分析

4. **订单详情/编辑/删除** — 当前只支持查看，不支持修改和删除已有订单

5. **批量操作** — 批量修改状态、批量删除

### 🟢 低优先级 / 锦上添花

6. **数据导出** — 从看板导出为 Excel / CSV
7. **数据对比** — 上月 vs 本月、去年 vs 今年
8. **定时任务** — 每天自动从马帮拉取数据（需影刀 RPA 配合）
9. **微信通知** — 每日出库报告推送到微信
10. **响应式优化** — 手机端查看体验改进
